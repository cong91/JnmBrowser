# Plan: Implement CSS Media Query Spoofing ("Gap Engine")

## Objective
Add `window.matchMedia` spoofing for both Camoufox and Chromium engines, filling the CSS media query fingerprinting gap identified in the audit.

## What We're Building
A `matchMedia` override injected into both browser engines that returns consistent spoofed results for common fingerprinting media queries:
- `prefers-color-scheme` (light/dark)
- `prefers-reduced-motion` (reduce/no-preference)
- `prefers-contrast` (no-preference/more/less)
- `prefers-reduced-data` (reduce/no-preference)
- `any-hover` (hover/none)
- `any-pointer` (fine/coarse)
- `color-gamut` (srgb/p3/rec2020)

## JS Injection Pattern (for both engines)
```javascript
// Inside the existing IIFE, append:
(function() {
  const originalMatchMedia = window.matchMedia.bind(window);
  const mediaOverrides = {
    "(prefers-color-scheme: dark)": { matches: false, media: "(prefers-color-scheme: dark)" },
    "(prefers-color-scheme: light)": { matches: true, media: "(prefers-color-scheme: light)" },
    "(prefers-reduced-motion: reduce)": { matches: false, media: "(prefers-reduced-motion: reduce)" },
    "(prefers-reduced-motion: no-preference)": { matches: true, media: "(prefers-reduced-motion: no-preference)" },
    // ... etc
  };
  window.matchMedia = function(query) {
    const override = mediaOverrides[query];
    if (override) {
      const mql = originalMatchMedia(query);
      Object.defineProperty(mql, "matches", { get: () => override.matches });
      Object.defineProperty(mql, "media", { get: () => override.media });
      return mql;
    }
    return originalMatchMedia(query);
  };
})();
```

The overrides object will be dynamically generated from config values (e.g., `media:prefersColorScheme: "dark"` → `prefers-color-scheme: dark = matches: true`).

## Edge cases handled
- Unknown media queries fall through to real `matchMedia`
- `addEventListener("change", ...)` still works on the spoofed MediaQueryList
- CSS `@media` rules are NOT affected (only JS `matchMedia` is spoofed — CSS media queries are handled by the browser rendering engine and are harder to spoof without browser-level patches)
- Chromium already has the fields in the TypeScript type; we just need to read them and inject
