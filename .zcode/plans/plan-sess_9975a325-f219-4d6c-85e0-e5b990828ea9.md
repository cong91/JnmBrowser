# Fix missing registration i18n keys

## Problem
UI is rendering **raw translation keys** because they are used in code but missing from locale files.

Confirmed broken keys (from screenshot + code audit):

| Key | Used in | Params |
|-----|---------|--------|
| `registration.cdkHint` | `src/components/account-registration-dialog.tsx:194` | `{count}`, `{total}` |
| `registration.storagePath` | `...:204` | — |
| `registration.storagePathHint` | `...:207` | — |
| `registration.startRegistrationWithCount` | `...:484` | `{total}` |

None of these exist in `en.json` (or any other locale). i18next falls back to the key path.

Also a small misuse:
- SMS validation toast uses `registration.smsServiceIdPlaceholder` (`e.g. 1`) as an error message (`account-registration-dialog.tsx:99`). Should be a real required-error key.

## Scope (keep tight)
- Fix **missing / wrong registration keys** that cause broken UI copy.
- Update **all 7 locales**: `en`, `es`, `fr`, `ja`, `pt`, `ru`, `zh` under `src/i18n/locales/`.
- Do **not** do a full re-translation of older English leftovers in non-en locales (many core labels are still English by design of prior partial work). Only add the missing keys + one required-error key.

## Implementation

### 1. Add missing keys to all locales under `registration`
Place near related keys (`cdkLabel`, `startRegistration`, storage block).

**English (`en.json`):**
```json
"cdkHint": "{count} CDK code(s) detected · {total} account(s) total",
"storagePath": "Account storage",
"storagePathHint": "App data / registered_accounts",
"startRegistrationWithCount": "Start Registration ({total})",
"smsServiceIdRequired": "Enter an SMS service ID"
```

**Other locales:** equivalent translations (not empty strings).

Suggested meaning for each:
- `cdkHint` — helper under CDK textarea showing parsed CDK count and planned account total
- `storagePath` — label above storage path box
- `storagePathHint` — short path hint (static; backend stores under `data_dir()/registered_accounts`)
- `startRegistrationWithCount` — start button when `totalAccounts > 0`
- `smsServiceIdRequired` — toast when SMS enabled but service ID empty

### 2. Fix wrong toast key in dialog
In `src/components/account-registration-dialog.tsx`:
```ts
// before
toast.error(t("registration.smsServiceIdPlaceholder"));
// after
toast.error(t("registration.smsServiceIdRequired"));
```

No other component changes required for the screenshot bugs.

### 3. Verify
- Confirm all 4 + 1 keys exist in every locale file (JSON valid).
- Grep `t("registration.` vs locale keys: no remaining missing among used registration keys for this dialog.
- Optional: open Auto Registration dialog and confirm labels no longer show `registration.cdkHint` / `registration.storagePathHint`.

## Out of scope
- Fetching real absolute storage path from Rust (current UI is static translated hint only).
- Hardcoded product names (`Chromium`, `Camoufox`) and CDK example placeholder.
- Full zh/es/… rewrite of pre-existing English registration strings.
- SMS dialog keys (`sms.*`) — already present in locales.

## Done criteria
- No raw `registration.cdkHint` / `storagePath` / `storagePathHint` / `startRegistrationWithCount` in UI.
- SMS empty service-id toast uses a real required message.
- All 7 locale files contain the new keys with non-empty values.
