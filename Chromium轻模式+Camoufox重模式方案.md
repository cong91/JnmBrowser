# Chromium 轻模式 + Camoufox 重模式方案

## 一、方案结论

JnmBrowser 后续采用双路线策略：

- **Chromium：轻模式**
- **Camoufox：重模式**

目标不是让两个内核都追求同样的能力深度，而是让它们各自承担最适合的职责，减少重复建设与维护成本。

---

## 二、定位拆分

### 1. Chromium（轻模式）

定位：

- 默认推荐浏览器
- 高兼容模式
- 快速创建与批量运行
- 自动化优先
- 指纹能力以“稳定、够用、低心智负担”为核心

适合场景：

- 日常使用
- 大批量 profile
- 自动化任务
- 需要 Chromium 生态兼容的网站
- 对深度手工指纹编辑要求不高的用户

### 2. Camoufox（重模式）

定位：

- 高级指纹模式
- 深度可控模式
- 更强原生指纹伪装能力
- 高级反检测实验场

适合场景：

- 需要更完整指纹编辑的用户
- 需要更强伪装能力的用户
- 需要高级环境模拟与精细调参的场景

---

## 三、Chromium 轻模式的能力边界

### 1. 保留能力

Chromium 后续保留并持续维护以下能力：

- `seed` 驱动的随机/稳定指纹
- 基础平台能力：
  - platform
  - platformVersion
  - brand
  - brandVersion
  - language
  - accept-language
  - timezone
  - hardwareConcurrency
- 代理能力
- 基础 WebRTC 策略
- 基础 GPU / WebGL 参数（按内核实际支持情况）
- MCP 自动化能力
- 高兼容浏览器运行能力

### 2. 不作为近期重点的能力

以下能力不再要求 Chromium 这条线优先做到完整可视化、完整编辑、完整显式回填：

- 完整 canvas 指纹编辑器
- 完整 audio 指纹编辑器
- 完整 fonts 枚举编辑器
- 完整 mediaDevices 细粒度编辑器
- 完整运行时指纹导出 UI
- 与 Wayfern 同等级的“完整指纹回读体验”

### 3. 设计原则

Chromium 轻模式强调：

- **简单可用**
- **批量稳定**
- **自动化友好**
- **兼容优先**
- **不把 UI 做成假完整配置器**

也就是说：

- 如果某些能力主要由 `seed` 在运行时隐式派生完成
- 则允许编辑页不展示为“完整静态字段”
- 但需要在产品说明中明确解释其行为

---

## 四、Camoufox 重模式的能力边界

### 1. 重点投入方向

Camoufox 后续作为高级模式，优先承接以下能力建设：

- 更完整的指纹字段编辑
- 更清晰的字体能力配置
- 更完整的 WebGL 相关配置
- 更强的地理位置 / 语言 / 时区联动能力
- 更完整的运行环境模拟
- 更适合作为“高级反检测”主阵地

### 2. 允许投入的高级方向

后续若继续增强，优先考虑放在 Camoufox：

- 更细粒度的 navigator / screen / media 能力编辑
- 更强的字体策略
- 更强的指纹模板库
- 更强的 profile 预设与分层配置
- 更深入的高级自动化兼容调优

---

## 五、MCP 自动化策略

### 1. Chromium

继续作为：

- MCP 自动化主力浏览器之一
- 高兼容自动化浏览器
- 默认自动化落地方向

### 2. Camoufox

继续保持：

- MCP 自动化可用
- 作为高级自动化浏览器存在
- 在需要更高指纹质量时使用

### 3. 策略结论

不是“只保留 Chromium 自动化，放弃 Camoufox 自动化”，而是：

- **两者都支持自动化**
- **Chromium 主打兼容/默认**
- **Camoufox 主打高级/深度**

---

## 六、产品层表达建议

### Chromium 对外文案建议

可表达为：

- 兼容模式
- 快速模式
- 随机指纹模式
- 适合批量任务与自动化

### Camoufox 对外文案建议

可表达为：

- 高级模式
- 深度指纹模式
- 更强环境模拟模式
- 适合高要求场景

---

## 七、后续开发优先级

### P0（当前路线直接执行）

1. 确认 Chromium 轻模式定位，不再追求 UI 假完整
2. 确认 Camoufox 重模式定位，作为高级指纹主线
3. 后续文档、说明、产品表达统一使用此策略

### P1（后续逐步优化）

1. 为 Chromium 编辑页增加说明：
   - 很多指纹能力由 `seed` 在运行时隐式派生
   - 空白字段不等于未生效
2. 优化 Chromium 轻模式文案与默认值
3. 继续提升 Camoufox 高级指纹编辑能力

### P2（未来条件允许时）

1. 视情况继续补充 fingerprint-chromium 深层能力接入
2. 视情况二开 fingerprint-chromium 导出运行时指纹
3. 视情况完善 Camoufox 高级探测/回显能力

---

## 八、最终决策原则

后续若遇到新功能，应先判断：

### 属于哪一类

#### A. 兼容性 / 批量 / 自动化 / 默认易用
优先放在 **Chromium**

#### B. 深度指纹 / 高级伪装 / 细粒度控制
优先放在 **Camoufox**

这样可以避免：

- 两条线重复建设
- Chromium 轻模式被做成复杂难维护的半成品
- Camoufox 的高级价值被稀释

---

## 九、一句话总结

JnmBrowser 后续采用：

- **Chromium 负责“简单、稳定、兼容、自动化”**
- **Camoufox 负责“深入、可控、高级、重指纹”**

这是当前阶段最省成本、最清晰、也最符合现有内核特性的路线。
